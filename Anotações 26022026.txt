Aula 2

BD/DB -> É uma coleção logicamente coerente dos dados com determinada significação intrínseca, com a finalidade de armazenamento organizado
de informações, otimizando sistemas, facilitando entradas, alterações, processamento e consulta de dados. Para sua criação e manutenção utiliza-se
o SGBD. O conjunto formado por BD + Aplicações que o manipulam é chamado Sistema de Banco de Dados. 

Principais Características do SGBD:
-Integridade: Impedir que um determinado código ou chave de uma tabela não tenha correspondência em outra. 
EX: O Código de uma disciplina na tabela 'Histórico Escolar' sem sua descrição na tabela 'disciplina'.

-Restrição ou Consistência: Armazenamento de informações centralizadas, mas com acesso descentralizado. A inconsistência ocorre quando um mesmo campo
tem valores diferentes em sistemas diferentes. Quando o armazenamento é centralizado este problema não ocorre.

-Segurança e Privacidade: Define para cada usuário o nível de acesso a ele concedido (leitura, leitura e gravação ou S/ acesso), a uma tabela ou campo 
específico. 

-Restauração ou Reorganização: Capacidade de se recuperar de falhas de Hardware e Software por meio de arquivos de Backup ou outros recursos.

Aula 3

Data Warehouse -> Armazém de dados, modelo dimensional, sistema projetado para análise de comportamento e tendências de negócio. Partem da implantação para a
necessidade do usuário. Idealizado por Innom/Kimball. 
	A modelagem dimensional é uma técnica de projeto lógico que busca apresentar dados em uma estrutura padronizada
e intuitiva que permite alto desempenho, evitando técnicas de normalização. Especificamente utilizada para suportar processamento analítico 
OLAP (On Line Analitic Process). 
	O tipo de negócio pode ser representado por um 'Cubo de Dados' -> Células do cubo: Valores -> Arestas do Cubo: Dimensões do dado.
É possível representar muitas dimensões, em geral de 4 a 15 modelos com + Dimensões podem ser combinadas.
